# Datasets for SERVE-SC

Real Solidity data for the SERVE-SC artifact. Drop this folder into the
repository root next to `serve_sc/`.

## What is here

| Path | Contents | Source | License |
|------|----------|--------|---------|
| `smartbugs-curated/` | 143 annotated vulnerable Solidity contracts (DASP-10) | [smartbugs/smartbugs-curated](https://github.com/smartbugs/smartbugs-curated) | **Apache-2.0** (bundled, see `smartbugs-curated/LICENSE`) |
| `ir/smartbugs_ir.jsonl` | the above converted to the SERVE-SC IR (detection labels) | generated by the converter | follows the dataset |
| `scripts/convert_smartbugs_to_ir.py` | SmartBugs -> IR converter | this package | MIT |
| `scripts/fetch_solidifi.py` | downloads SolidiFI (not bundled) | this package | MIT |

SmartBugs Curated is redistributed here under Apache-2.0 with its `LICENSE` and
attribution preserved (`NOTICE`). It is the same dataset cited in the paper.

## Why SolidiFI is not bundled

SolidiFI-benchmark did not advertise a redistribution license at packaging time,
so re-hosting it would be unsafe. Run `python scripts/fetch_solidifi.py` to pull
it from its source, and **check its license before adding it to a public repo.**
Your 127-case financial-service suite is yours to release; place it under
`data/financial-suite/` in the IR format and cite its provenance.

## Taxonomy mapping (a modelling decision you own)

SmartBugs uses the DASP-10 taxonomy; the paper uses five classes. The converter
maps them as below. **Edit `CATEGORY_TO_CLASS` in the converter to match your
paper**, then re-run.

| SmartBugs category | -> paper class | contracts |
|--------------------|----------------|-----------|
| reentrancy | reentrancy | 31 |
| access_control | access_control | 18 |
| unchecked_low_level_calls | unchecked_calls | 52 |
| arithmetic | logic_inconsistency | 15 |
| front_running | logic_inconsistency | 4 |
| bad_randomness | logic_inconsistency | 8 |
| time_manipulation | logic_inconsistency | 5 |
| denial_of_service | (dropped) | 6 |
| short_addresses | (dropped) | 1 |
| other | (dropped) | 3 |

Two caveats stated plainly:

1. **No oracle data.** SmartBugs Curated has no `oracle_misuse` category, so that
   paper class has zero examples here. Supply oracle-misuse contracts from
   elsewhere (e.g. your financial-service suite) or the class stays empty.
2. **Coarse merges.** Folding `front_running`, `bad_randomness`, and
   `time_manipulation` into `logic_inconsistency` is a simplification; if your
   paper treats these differently, change the mapping.

By default the converter **drops** the 10 contracts whose category has no clean
class (use `--keep-unmapped` to keep them with all-zero labels).

## What the IR does and does not contain

The converter fills, per contract:

- `labels` — the real five-class label from the dataset annotations;
- `functions` — regex-extracted names (approximate);
- `static_cues` — a few **approximate** regex cues (low-level calls,
  `delegatecall`, `selfdestruct`, `require` count, ...).

It does **not** fill `edges`, `tx_sequences`, `state_vars`, `ext_calls`,
`oracles` (need a Solidity front-end / AST + call graph), nor `phi`, `service`,
`impact` (need exploit-precondition extraction and on-chain/DeFi annotations).
So the output supports the **Stage-1 detection** task with real labels; for the
full pipeline, populate those fields from your front-end. The node features in
`graph.py` are placeholders until you attach real opcode/AST embeddings.

## Usage

```bash
# regenerate the IR (after editing the mapping if needed)
python scripts/convert_smartbugs_to_ir.py \
    --dataset smartbugs-curated --out ir/smartbugs_ir.jsonl

# load it (each line is one IR dict)
python -c "import json; rows=[json.loads(l) for l in open('ir/smartbugs_ir.jsonl')]; print(len(rows), 'contracts', sum(any(r['labels'].values()) for r in rows), 'vulnerable')"
```

To train detection on real labels you must attach real node features (and ideally
the static graph edges) in `serve_sc/graph.py`; with the placeholder features the
detector cannot learn from this data.
